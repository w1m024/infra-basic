# 模型精度与量化

## 什么是模型精度

模型精度指的是权重和计算使用的数值位数。位数越多越精确，但占用的显存和计算量也越大。

### 浮点精度

| 精度 | 位数 | 说明 |
|---|---|---|
| FP32 | 32 位 | 单精度浮点，训练默认精度，显存占用大 |
| FP16 | 16 位 | 半精度浮点，推理常用，显存减半 |
| BF16 | 16 位 | Brain Float 16，范围比 FP16 大，训练更稳定 |
| FP8 | 8 位 | 新一代低精度，Hopper/Blackwell GPU 支持 |

### 整数量化

| 精度 | 位数 | 说明 |
|---|---|---|
| INT8 | 8 位 | 权重量化，显存减 4 倍（相比 FP32） |
| INT4 | 4 位 | 权重量化，显存减 8 倍，质量损失明显 |

## 精度对比

| 精度 | 每参数字节 | 7B 模型显存 | 训练 | 推理 | 质量 |
|---|---|---|---|---|---|
| FP32 | 4 | ~28 GB | ✓ | ✓ | 最高 |
| FP16/BF16 | 2 | ~14 GB | ✓ | ✓ | 接近 FP32 |
| INT8 | 1 | ~7 GB | ✗ | ✓ | 轻微损失 |
| INT4 | 0.5 | ~3.5 GB | ✗ | ✓ | 明显损失 |

## 什么是量化

把模型权重从高精度（如 FP16，16位）转换成低精度（如 INT4，4位）。

**目的**：用精度换显存和速度。

**例子**：一个 7B 参数的模型
- FP16：7B × 2字节 = 14 GB
- INT4：7B × 0.5字节 = 3.5 GB

**代价**：精度损失，模型可能更容易出错或"胡说八道"。

**常用方法**：GPTQ、AWQ、GGUF（llama.cpp 用的格式）。

## 量化方法

### 训练后量化（PTQ）

模型训练完成后直接量化，不需要重新训练：

- **GPTQ**：基于二阶信息的权重量化，效果好但速度慢
- **AWQ**：激活感知的权重量化，保留重要通道的精度
- **GGUF**：llama.cpp 使用的量化格式，支持多种量化级别

### 量化感知训练（QAT）

训练时模拟量化效果，精度损失更小，但需要更多训练资源。

## 选择建议

| 场景 | 推荐精度 | 原因 |
|---|---|---|
| 聊天/问答 | INT4 | 速度优先，质量可接受 |
| 代码生成 | INT8 | 减少幻觉，更稳定 |
| 长文生成 | FP16 | 避免精度累积误差 |
| 训练/微调 | FP16/BF16 | 必须保持精度 |
| 生产部署 | INT8/FP16 | 平衡速度和质量 |

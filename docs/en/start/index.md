# Getting Started

This path is organized as: "Run it → Measure it → Explain it → Read source → Modify code".

## Recommended Order

1. Learning Roadmap - Build the full map first.
2. Environment & CUDA - Confirm your runtime environment.
3. One Month Plan - Compress the roadmap into an executable cycle.
4. Systems - Understand requests, KV cache, scheduling, and benchmarks.
5. Source Reading Map - Enter vendor project source code.

## Validation Output

| Phase | Output |
|---|---|
| Environment | PyTorch CUDA inference record |
| System | Request main flow diagram |
| Lab | TTFT / TPOT / tokens/s benchmark |
| Source | At least two source reading maps |
| Contribution | Issue reproduction, docs, test, or minimal PR |

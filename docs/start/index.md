# 从零开始

这条路线按“能跑通 → 能测量 → 能解释 → 能读源码 → 能改代码”组织。

## 推荐顺序

1. [学习路线](./roadmap.md)：先建立完整地图。
2. [环境与 CUDA 路线](./environment.md)：确认默认运行环境。
3. [一月计划](./one-month-plan.md)：把路线压缩为一个可执行周期。
4. [系统原理](/systems/)：理解请求、KV cache、调度和 benchmark。
5. [源码阅读地图](/source-reading/)：进入 vendor 项目源码。

## 验收产出

| 阶段 | 产出 |
|---|---|
| 环境 | PyTorch CUDA 推理记录 |
| 系统 | 请求主流程图 |
| 实验 | TTFT / TPOT / tokens/s benchmark |
| 源码 | 至少两篇源码地图 |
| 贡献 | issue reproduction、docs、test 或最小 PR |
